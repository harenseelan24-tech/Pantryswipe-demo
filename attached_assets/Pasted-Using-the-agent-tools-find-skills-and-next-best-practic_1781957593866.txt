=== ABOUT THIS APP ===

PantrySwipe is a Tinder-style AI cooking ecosystem built as an Expo mobile app. Here is the full structure:

WORKSPACE ROOT:
- pnpm workspaces (pnpm-workspace.yaml)
- Node.js 24, TypeScript 5.9
- Root scripts: `pnpm run typecheck` (runs typecheck:libs then all artifact typechecks)

MOBILE APP: artifacts/pantryswipe/
- Expo SDK 54, expo-router v6 (file-based routing), React Native 0.81
- State: React Context + AsyncStorage via artifacts/pantryswipe/context/AppContext.tsx
- Gestures: PanResponder (NOT gesture-handler) — do not change this
- UI: @expo/vector-icons (Feather), react-native-reanimated, expo-haptics
- NO backend DB required — all state lives in AsyncStorage

SCREENS (artifacts/pantryswipe/app/):
- app/index.tsx — root guard: checks AsyncStorage, routes to /welcome or /(tabs)
- app/welcome.tsx — first launch welcome screen
- app/onboarding.tsx — onboarding flow
- app/(tabs)/ — 5-tab navigation: index (Discover/Home), pantry, planner, social, profile
- app/recipe/[id].tsx — single recipe detail screen
- app/ai-chef.tsx — AI Chat interface with pantry-aware mock responses + quick prompt chips
- app/party-planner.tsx — multi-step wizard: event type, guest count, serving style, budget → menu + timeline
- app/notifications.tsx — notifications screen
- Stack screens for recipe/[id], ai-chef, party-planner, notifications

KEY COMPONENTS:
- artifacts/pantryswipe/components/SwipeCard.tsx — Tinder-style swipe card using PanResponder
- artifacts/pantryswipe/context/AppContext.tsx — global state: pantry items, recipes, profile, stats, cooking streak, XP/levels
- artifacts/pantryswipe/data/mockData.ts — all recipe, pantry, social mock data
- artifacts/pantryswipe/constants/colors.ts — brand palette (light + dark)
