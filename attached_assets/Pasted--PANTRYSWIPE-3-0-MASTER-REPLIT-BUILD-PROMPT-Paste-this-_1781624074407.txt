## OVERVIEW

Build a full-stack mobile-first web application called **PantrySwipe** — an AI-powered cooking ecosystem that combines pantry management, Tinder-style recipe discovery, meal planning, nutrition tracking, food waste reduction, barcode scanning, and a social feed into one beautifully designed app.

**Tech Stack:**
- Frontend: React (Vite) + TypeScript + Tailwind CSS
- Backend: Node.js + Express + TypeScript
- Database: PostgreSQL with Drizzle ORM
- AI: Anthropic Claude API (`claude-sonnet-4-6`) for recipe generation, macro adjustment, shopping list generation
- Barcode: `@zxing/browser` for in-browser camera barcode scanning (no native SDK needed)
- Auth: Supabase Auth or simple JWT with bcrypt
- Storage: Supabase Storage (for user photos, food images)
- Package manager: pnpm

**Design System:**
- Color palette: White (`#FFFFFF`), Soft Blue (`#4A90D9`), Light Blue (`#EBF4FF`), Warm Off-White (`#F8F9FA`), Dark Navy (`#1A2E4A`), Accent Teal (`#2DD4BF`), Warning Amber (`#F59E0B`), Danger Red (`#EF4444`), Success Green (`#10B981`)
- Typography: `Inter` (UI) + `Playfair Display` (headings/recipe names) — both from Google Fonts
- Corner radius: `rounded-2xl` everywhere (homey, soft, not clinical)
- Cards: white with subtle `shadow-md`, 1px blue-tinted border
- Bottom navigation bar: 5 icons, pill-shaped active indicator in soft blue
- Micro-animations: card swipe physics, ingredient entry animations, loading skeletons
- Icons: Lucide React
- The overall feel: like a clean modern kitchen — warm, organised, inviting. Think Notion meets a high-end recipe book.

## AI INTEGRATION (Claude API)

Use `claude-sonnet-4-6` for all AI tasks.

**1. Recipe Generation from Pantry:**

```
System: You are a professional chef and nutritionist. Generate recipes that are realistic, delicious, and achievable for home cooks. Always return valid JSON only.

User: I have these pantry items: [list]. My dietary restrictions: [list]. Cooking skill: [level]. Occasion: [occasion]. Generate 10 recipe cards.

Response format: JSON array of recipe objects with fields: name, cuisine, difficulty, cook_time_mins, servings, calories, macros {protein, carbs, fat, fibre}, ingredients [{name, quantity, unit, inPantry: true/false}], steps [{step_number, instruction, timer_seconds (if applicable)}], image_prompt (for placeholder), tags, event_types
```

**2. Macro Adjustment:**

```
User: I want to increase the protein in [recipe name] from 25g to 45g per serving. Keep the dish the same cuisine and style. Adjust ingredients accordingly and recalculate all macros.

Response: Updated ingredients JSON + updated macros JSON
```

**3. Shopping List Generation:**

```
User: Based on my meal plan for [date range] with these meals: [meal list], and my current pantry: [pantry JSON], generate a categorised shopping list for everything I need to buy. Group by supermarket aisle category. Include estimated costs in [currency].

Response: JSON shopping list categorised by section
```

**4. Expiry Recipe Suggestion:**

```
User: These pantry items are expiring soon: [list with expiry dates]. Suggest 5 recipes that use as many of these items as possible. Prioritise the most critical items.

Response: Array of 5 recipe suggestions with urgency explanation
```

**5. Nutrition Analysis:**

```
User: Here is my meal log for this week: [JSON]. Analyse my nutrition patterns, highlight deficiencies or excesses, and suggest 3 actionable improvements for next week.

Response: Analysis text + 3 specific suggestions
```

## BARCODE SCANNER IMPLEMENTATION

Use `@zxing/browser` (install: `npm install @zxing/browser`).

```typescript
// BarcodeScanner.tsx
import { BrowserMultiFormatReader } from '@zxing/browser';

const scanner = new BrowserMultiFormatReader();

// Request camera permission
const stream = await navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment' } });

// Start continuous scanning
scanner.decodeFromVideoElement(videoElement, (result, error) => {
  if (result) {
    const barcode = result.getText();
    fetchProductByBarcode(barcode);
  }
});

// Fetch from Open Food Facts
async function fetchProductByBarcode(barcode: string) {
  const res = await fetch(`https://world.openfoodfacts.org/api/v0/product/${barcode}.json`);
  const data = await res.json();
  if (data.status === 1) {
    const product = data.product;
    return {
      name: product.product_name,
      brand: product.brands,
      calories: product.nutriments['energy-kcal_100g'],
      protein: product.nutriments.proteins_100g,
      carbs: product.nutriments.carbohydrates_100g,
      fat: product.nutriments.fat_100g,
      image: product.image_url,
    };
  }
}
```

Camera permission UI: full-screen camera view, targeting reticle overlay (rounded square), animated scan line, "Point at barcode" instruction, manual entry fallback button at bottom.

## FINAL NOTES FOR REPLIT AGENT

1. Build mobile-first. All layouts should be optimised for 375px–430px width first.
2. Use Tailwind CSS throughout. No inline styles except for dynamic values (e.g., swipe transform).
3. All text must be legible — minimum 14px body, 18px headings. Never use light grey text on white.
4. Every interactive element must have hover + active states.
5. The swipe physics must feel natural — use CSS `transition: transform 0.3s cubic-bezier(0.25, 0.46, 0.45, 0.94)` for card throw.
6. The app should work fully in the browser without a native app wrapper — no Expo, no React Native.
7. All Claude AI API calls should go through the backend (server/api/ai.ts) — never expose API keys to the frontend.
8. Use environment variables: `DATABASE_URL`, `ANTHROPIC_API_KEY`, `SUPABASE_URL`, `SUPABASE_ANON_KEY`.
9. Include a `/health` endpoint on the backend.
10. Include a `README.md` with setup instructions and feature list.

Start by building the database schema, then the backend API routes, then the frontend screens in this order: Welcome → Onboarding → Home → Pantry → Swipe → Recipe Detail → Planner → Social. Then wire up the AI features last.

**The defining goal of this app:** Every user who opens it should feel like they have a personal chef living in their phone — one who knows exactly what's in their fridge, what they feel like eating tonight, and won't let anything go to waste.
