# MASTER APP-BUILDER PROMPT — PANTRYSWIPE
### Version 2.0 — Full UX/UI + Design System + AI Behavior Spec

Build a production-ready mobile-first application called **PantrySwipe** for iOS, Android, and Web.

PantrySwipe is an AI-powered cooking ecosystem. It is not a recipe app. It is the answer to one universal question:

> **"What can I cook right now, with what I already have?"**

The experience should feel like a seamless blend of Tinder's swipe decisions, TikTok's discovery feed, Spotify's mood-based filtering, MyFitnessPal's nutrition intelligence, and a personal AI chef that knows your kitchen better than you do.

---

## DESIGN SYSTEM & VISUAL IDENTITY

### Philosophy
Food is emotional. The design must feel warm, alive, and indulgent — like a well-lit kitchen at golden hour, not a sterile health app. Every screen should make the user want to cook *right now*.

### Color Palette
- **Primary Background (Light Mode):** `#FAFAF8` — warm off-white, like parchment
- **Primary Background (Dark Mode):** `#141210` — deep espresso, not pure black
- **Surface Cards:** `#FFFFFF` light / `#1E1B18` dark
- **Brand Accent — Saffron:** `#F5A623` — used for CTAs, highlights, active states
- **Secondary Accent — Herb Green:** `#4CAF76` — used for "match" states, freshness indicators
- **Destructive / Skip:** `#E84040` — swipe left, discard
- **Save / Bookmark:** `#5B8EF5` — swipe up, save for later
- **Text Primary:** `#1A1A1A` light / `#F0EDE8` dark
- **Text Secondary:** `#6B6B6B` light / `#9E9E9E` dark
- **Divider:** `#EBEBEB` light / `#2A2724` dark

### Typography
- **Display / Hero:** `Fraunces` (variable, optical size) — serif, warm, editorial. Used for recipe titles, hero moments.
- **Body / UI:** `Inter` (variable) — clean, legible, neutral. Used for instructions, labels, body text.
- **Accent / Badges:** `Space Grotesk` — geometric, modern. Used for tags, calorie counts, timers, numeric data.
- **Type Scale:**
  - Hero: 36px / 700 weight / -1px tracking
  - Title: 24px / 600
  - Subtitle: 18px / 500
  - Body: 15px / 400 / 1.6 line height
  - Caption / Label: 12px / 500 / uppercase + 0.5px tracking

### Iconography
Use `Phosphor Icons` (duotone variant). Never mix icon sets. Icons must always have a label on first use.

### Motion & Animation
- Swipe cards: physics-based spring animation (react-spring or Framer Motion). Cards should feel like physical cards with weight.
- Screen transitions: slide-up for detail views, fade for tab switches.
- Skeleton loaders: always, no spinners.
- Micro-interactions: subtle haptic-paired animations on swipe decisions (bounce right for yes, tilt left for skip, float up for save).
- Reduce motion: respect `prefers-reduced-motion` system setting fully.

### Component Language
- Border radius: `16px` cards, `12px` inputs, `100px` pills/chips, `8px` small elements
- Shadows: `0 2px 12px rgba(0,0,0,0.08)` cards light mode; `0 2px 16px rgba(0,0,0,0.4)` dark mode
- Spacing unit: `8px` base grid
- Image treatment: always full-bleed on cards, `object-fit: cover`, darkened gradient overlay at bottom for text legibility
- Bottom safe area: always respect iOS/Android notch and home indicator

## SMART PANTRY SYSTEM — DATA SPEC

```
pantry_items table:
- pantry_id (uuid)
- user_id (foreign key)
- ingredient_name (string)
- quantity (float)
- unit (enum: g, kg, ml, L, cups, tbsp, tsp, pieces, packs)
- purchase_date (timestamp)
- expiry_date (timestamp, nullable)
- estimated_cost (float, nullable)
- category (enum: Produce, Meat, Dairy, Dry Goods, Frozen, Condiments, Beverages, Spices, Other)
- storage_location (enum: Fridge, Freezer, Pantry, Counter)
- status (enum: Fresh, Use Soon, Expiring, Expired)
- source (enum: Manual, Receipt Scan, Camera Scan, Recipe Cook, Grocery Import)
- last_updated (timestamp)
```

**Auto-update triggers:**
1. After cooking: deduct exact recipe quantities used, scaled to servings made
2. After receipt scan: add new items, increment existing item quantities
3. After camera scan: create or update items with confidence score > 70%
4. Manual edit: user direct edit
5. Expiry AI estimate: if no expiry entered, AI estimates based on ingredient type and category

**Post-cooking flow:**
- Prompt: `"How many servings did you make?"` (stepper)
- Prompt: `"Did you use the full amounts?"` (toggle, default YES)
- If no: per-ingredient sliders to adjust actual usage
- Pantry updates instantly
- Show: `"Pantry updated. You have 250g chicken left."` toast

## AI COMPUTER VISION — FRIDGE/PANTRY SCANNER

- User opens camera in-app
- Real-time overlay shows detected ingredients as bounding boxes
- Confidence score shown per detection: `"Broccoli 94%" · "Chicken (raw) 88%"`
- Detects: vegetables, fruits, raw meat, cooked dishes, dairy, beverages, condiments, sauces, packaged goods (barcode fallback)
- After capture: review screen shows all detections as editable chips
  - User can: confirm, correct name, adjust quantity, delete
- CTA: `"Add X items to pantry"` — batch confirmation
- Edge case: `"Not sure what this is?"` — tap to manually type

## RECEIPT SCANNER

- Shows parsed list for review before adding
- Estimated expiry auto-suggested per item type
- Adds directly to pantry with `source: Receipt Scan`

## RECIPE RECOMMENDATION ENGINE — RANKING LOGIC

```
Recommendation Score =
  Ingredient Match Score (40% weight)
  × Preference Score (25% weight)
  × Nutrition Score (20% weight)
  × Popularity Score (10% weight)
  × Freshness Urgency Score (5% weight — boost recipes using expiring ingredients)
```

**Ingredient Match Score:** % of recipe ingredients found in pantry
**Preference Score:** cuisine preference + dietary compliance + skill level match
**Nutrition Score:** closeness to user's daily nutrition goals
**Popularity Score:** global rating × recency weighted
**Freshness Urgency Score:** recipes using the user's soonest-expiring ingredients are boosted
