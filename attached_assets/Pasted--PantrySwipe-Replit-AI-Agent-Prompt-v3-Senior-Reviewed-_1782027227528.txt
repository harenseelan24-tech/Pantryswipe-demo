## SECTION 4 — SWIPE CARD (CRITICAL) ⚠️ HIGH RISK

File: `artifacts/pantryswipe/components/SwipeCard.tsx`

**Risk:** PanResponder + Animated has many silent failure modes — wrong `useNativeDriver` usage crashes at runtime, gesture interception breaks child button taps, and animation conflicts between translate and opacity are common. Follow each sub-section exactly.

### 4.0 — Pre-edit safety check (do this FIRST)
Read the entire existing `SwipeCard.tsx` and document what it currently does. Identify which of the items below are already correctly implemented vs. missing/broken. Only change what is broken — do not rewrite working code.

### 4.1 — PanResponder setup
- `onStartShouldSetPanResponder` must return `false` — do not intercept taps, only drags.
- `onMoveShouldSetPanResponder` must return `true` ONLY when horizontal movement exceeds vertical movement:
  ```typescript
  onMoveShouldSetPanResponder: (_, gestureState) =>
    Math.abs(gestureState.dx) > Math.abs(gestureState.dy) && Math.abs(gestureState.dx) > 5,
  ```
  This prevents the swipe card from blocking vertical scrolls in parent ScrollViews and allows buttons inside the card to receive taps.
- `onPanResponderMove` must use `Animated.event`:
  ```typescript
  onPanResponderMove: Animated.event(
    [null, { dx: pan.x, dy: pan.y }],
    { useNativeDriver: false }  // REQUIRED — Animated.event does not support native driver
  ),
  ```
- `onPanResponderRelease` must evaluate thresholds and call the correct callback:
  ```typescript
  onPanResponderRelease: (_, gestureState) => {
    const { dx, dy } = gestureState;
    if (dx > 100) {
      dismissCard('cook');
    } else if (dx < -100) {
      dismissCard('skip');
    } else if (dy < -100) {
      dismissCard('save');
    } else {
      // Below threshold — snap back to centre
      Animated.spring(pan, { toValue: { x: 0, y: 0 }, useNativeDriver: false }).start();
    }
  },
  ```
- **Mitigation:** Add a temporary `console.log('[SwipeCard] released dx:', gestureState.dx, 'dy:', gestureState.dy)` inside `onPanResponderRelease` to verify thresholds during testing. Remove after confirming.

### 4.2 — Card rotation and transform
- Interpolate `pan.x` to rotation:
  ```typescript
  const rotation = pan.x.interpolate({
    inputRange: [-200, 0, 200],
    outputRange: ['-15deg', '0deg', '15deg'],
    extrapolate: 'clamp',
  });
  ```
- Apply ALL transforms in a single transform array — never split translate and rotate across separate style props:
  ```typescript
  style={[styles.card, {
    transform: [
      { translateX: pan.x },
      { translateY: pan.y },
      { rotate: rotation },
    ]
  }]}
  ```
- Use `useNativeDriver: false` on this Animated.View — translation and rotation together cannot use the native driver when the Animated.Value is also used with `Animated.event`.

### 4.3 — Overlay labels
Overlay badges must fade in as the user drags, well before the threshold is reached so they know what action they're triggering:

- COOK badge (green, drag right):
  ```typescript
  opacity: pan.x.interpolate({ inputRange: [20, 100], outputRange: [0, 1], extrapolate: 'clamp' })
  ```
- SKIP badge (red, drag left):
  ```typescript
  opacity: pan.x.interpolate({ inputRange: [-100, -20], outputRange: [1, 0], extrapolate: 'clamp' })
  ```
- SAVE badge (blue, drag up):
  ```typescript
  opacity: pan.y.interpolate({ inputRange: [-100, -20], outputRange: [1, 0], extrapolate: 'clamp' })
  ```
- All badge Animated.Views must use `useNativeDriver: false` — they are driven by the same `pan` value that uses `useNativeDriver: false`.

**Note on SKIP and SAVE ranges:** The outputRange `[1, 0]` with ascending inputRange means: at the far-left/far-up value the badge is fully opaque, and as the drag returns toward 0 it fades out. This is correct — the badge is visible when dragging in the right direction and fades as the card returns to centre.

import { View, ActivityIndicator } from 'react-native';
import { router, useRootNavigationState } from 'expo-router';
import AsyncStorage from '@react-native-async-storage/async-storage';

export default function Index() {
  const navState = useRootNavigationState();
  const [checked, setChecked] = useState(false);

  useEffect(() => {
    // Don't navigate until the navigator is ready
    if (!navState?.key) return;

    const checkOnboarding = async () => {
      try {
        const value = await AsyncStorage.getItem('pantryswipe_onboarding_complete');
        if (value === 'true') {
          router.replace('/(tabs)');

          router.replace('/welcome');

        router.replace('/welcome');

      }

### 11.2 — Cooking streak (use this exact logic)
The streak VALUE and the last-cooked DATE are two separate things and must both be persisted.

AsyncStorage keys to use consistently:
- `'pantryswipe_last_cooked_date'` — ISO date string `"YYYY-MM-DD"`
- `'pantryswipe_streak'` — integer stored as a string, e.g. `"7"`

Call this function inside `markRecipeCooked` in AppContext (after marking the recipe cooked):
```typescript
const updateStreak = async () => {
  const todayUTC = new Date().toISOString().split('T')[0]; // e.g. "2026-06-21"
  const lastCookedDate = await AsyncStorage.getItem('pantryswipe_last_cooked_date');
  const storedStreak = await AsyncStorage.getItem('pantryswipe_streak');
  const currentStreak = storedStreak ? parseInt(storedStreak, 10) : 0;

  let newStreak: number;

  if (!lastCookedDate) {
    newStreak = 1; // First cook ever
  } else {
    const diffMs = new Date(todayUTC).getTime() - new Date(lastCookedDate).getTime();
    const diffDays = Math.round(diffMs / (1000 * 60 * 60 * 24));

    if (diffDays === 0) {
      newStreak = currentStreak; // Already cooked today — no change
    } else if (diffDays === 1) {
      newStreak = currentStreak + 1; // Consecutive day
    } else {
      newStreak = 1; // Streak broken — restart from 1
    }
  }

  // Only write date if this is a new day's cook
  if (lastCookedDate !== todayUTC) {
    await AsyncStorage.setItem('pantryswipe_last_cooked_date', todayUTC);
  }
  await AsyncStorage.setItem('pantryswipe_streak', String(newStreak));

  // Update context so UI reflects immediately
  setProfile(prev => ({ ...prev, streak: newStreak }));
};
```

### 12.3 — useNativeDriver rules
- `Animated.event` with PanResponder: **always `useNativeDriver: false`** — no exceptions.
- `Animated.timing` / `Animated.spring` on transform or opacity: `useNativeDriver: true` — BUT only if the Animated.Value is not also used in `Animated.event` or on a layout prop. If shared with PanResponder, it must remain `false`.
- `Animated.timing` on width, height, backgroundColor, or any layout prop: **always `useNativeDriver: false`**.
- **Never mix** `useNativeDriver: true` and `false` on the same `Animated.Value` — pick one and use it everywhere for that value.
