CAUSE A — Broken import in the account-created screen file:

CAUSE B — Missing route definition:
  expo-router creates routes from the file system.
  If the navigation call does:
    router.push('/account-created')
  But the file lives at a different path than expo-router
  expects, the route load fails and crashes the app.

  Verify:
    - The file path matches the route string exactly
    - If using route groups like (auth), the navigation call
      must include the group: router.push('/(auth)/account-created')
      OR the group must be transparent (not in the nav path)
    - Check _layout.tsx in the relevant folder for route config

CAUSE C — Unhandled promise rejection during account creation:
  The signup/register function (called before navigation) may
  throw an error that is not caught, crashing the JS thread.

  Find the function that:
    1. Creates the user account (API call, AsyncStorage, etc.)
    2. Then navigates to account-created screen

  Check if it has try/catch:

  WRONG (crashes on any error):
    const handleSignup = async () => {
      await createAccount(email, password);
      router.push('/account-created');
    };

  RIGHT (catches errors safely):
    const handleSignup = async () => {
      try {
        await createAccount(email, password);
        router.push('/account-created');
      } catch (error) {
        console.error('Signup failed:', error);
        setError('Account creation failed. Please try again.');
      }
    };

  Add try/catch to every async function in the auth flow.

CAUSE D — State update after navigation (memory leak crash):
  If the signup screen calls setState after navigating away,
  React will throw: "Can't perform a React state update on an
  unmounted component."

  Look for patterns like:
    const handleSignup = async () => {
      router.push('/account-created');  // navigates away
      await doSomething();              // still running
      setLoading(false);               // crashes — component gone
    };

  Fix by using a mounted ref:
    const isMounted = useRef(true);
    useEffect(() => {
      return () => { isMounted.current = false; };
    }, []);

    const handleSignup = async () => {
      try {
        await createAccount(email, password);
        if (isMounted.current) {
          router.push('/account-created');
        }
      } catch (e) {
        if (isMounted.current) setError(e.message);
      }
    };

CAUSE E — AppContext state update on account creation:
  If the signup function calls a context setter (e.g.
  setUser(), setIsAuthenticated()) and the context update
  triggers a re-render that crashes due to undefined data:

  Find in AppContext.tsx: what happens when a user is set.
  Check if any component reads user.name, user.avatar, etc.
  without null-checking. If user is set to a partial object
  during signup, accessing user.name crashes immediately.

  Fix by null-checking every AppContext field access:
  WRONG: <Text>{user.name}</Text>
  RIGHT: <Text>{user?.name ?? 'User'}</Text>

CAUSE F — Navigation to tabs while auth redirect is active:
  expo-router may have a redirect configured in _layout.tsx
  that sends unauthenticated users back to the auth screen.
  After account creation, if isAuthenticated is set to true
  AND there's a redirect check in the same render cycle,
  two competing navigations can crash the router.

  Look in _layout.tsx for:
    useEffect(() => {
      if (!isAuthenticated) router.replace('/login');
    }, [isAuthenticated]);

  If this exists alongside the post-signup navigation,
  they conflict. Fix by using router.replace() for signup
  completion (not router.push()) so the auth screen is removed
  from the stack, preventing the redirect from firing on it.

Fix 3a — Wrap the entire account-created screen in an error
boundary to prevent a single component crash from killing
the whole app. Create this file if it doesn't exist:

  File: artifacts/pantryswipe/components/ErrorBoundary.tsx

  import React from 'react';
  import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

  interface Props {
    children: React.ReactNode;
    fallback?: React.ReactNode;
  }

  interface State {
    hasError: boolean;
    error: Error | null;
  }

  export class ErrorBoundary extends React.Component<Props, State> {
    constructor(props: Props) {
      super(props);
      this.state = { hasError: false, error: null };
    }

    static getDerivedStateFromError(error: Error): State {
      return { hasError: true, error };
    }

    componentDidCatch(error: Error, info: React.ErrorInfo) {
      console.error('ErrorBoundary caught:', error, info);
    }

    render() {
      if (this.state.hasError) {
        return this.props.fallback ?? (
          <View style={styles.container}>
            <Text style={styles.title}>Something went wrong</Text>
            <Text style={styles.message}>
              {this.state.error?.message ?? 'Unknown error'}
            </Text>
            <TouchableOpacity
              style={styles.button}
              onPress={() => this.setState({ hasError: false, error: null })}
            >
              <Text style={styles.buttonText}>Try Again</Text>
            </TouchableOpacity>
          </View>
        );
      }
      return this.props.children;
    }
  }

  const styles = StyleSheet.create({
    container: {
      flex: 1,
      alignItems: 'center',
      justifyContent: 'center',
      padding: 24,
      backgroundColor: '#FFF8F4',
    },
    title: {
      fontSize: 20,
      fontWeight: '700',
      color: '#141210',
      marginBottom: 12,
    },
    message: {
      fontSize: 14,
      color: '#7A7570',
      textAlign: 'center',
      marginBottom: 24,
    },
    button: {
      backgroundColor: '#F5A623',

    },
