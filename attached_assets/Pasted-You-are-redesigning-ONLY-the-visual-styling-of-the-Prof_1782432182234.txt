CRITICAL TRANSLATION RULES
(The design came from a web HTML file — these rules prevent breakage)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

RULE 1 — NO WEB-ONLY CSS
These Tailwind/CSS properties do not exist in React Native and must
NEVER appear in StyleSheet:
  backdrop-filter, blur(), brightness(), backdrop-blur-xl, blur-md,
  bg-gradient-to-t, transition, :active, :hover, window.*, pageYOffset,
  classList, document.getElementById, env(safe-area-inset-bottom)

RULE 2 — NO MATERIAL SYMBOLS ICONS
The design uses Google Material Symbols webfont. The app uses
@expo/vector-icons Feather exclusively. Use this mapping:
  arrow_back        → Feather "arrow-left"
  settings          → Feather "settings"
  stars             → Feather "star"  (filled variant via color)
  restaurant        → Feather "coffee"  (closest food icon)
  local_fire_department → Feather "zap"
  eco               → Feather "sun"
  emoji_events      → Feather "award"
  bakery_dining     → Feather "gift"
  wine_bar          → Feather "droplet"
  person            → Feather "user"
  calendar_month    → Feather "calendar"
  explore           → Feather "compass"

RULE 3 — NO EXTERNAL IMAGE URLS
The Stitch HTML uses lh3.googleusercontent.com URLs. Do NOT use these.
  - Avatar image: use the field from AppContext (e.g. user.avatar). If
    the field is a URI string, use it in source={{ uri: value }}. If it
    doesn't exist, use a placeholder View with the user's initials.
  - Recipe grid images: use the image field from the savedRecipes /
    cookedRecipes arrays in AppContext. Use require() if they are local
    assets, or source={{ uri: value }} if they are URI strings. Match
    whatever format mockData.ts uses.

RULE 4 — NO DOM-BASED TAB SWITCHING
The Stitch JS uses switchTab() with classList. Replace entirely with:

  const [activeTab, setActiveTab] = useState<'saved' | 'cooked'>('saved');

  Tab button: onPress={() => setActiveTab('saved')}
  Content: {activeTab === 'saved' ? <SavedGrid /> : <CookedGrid />}

  Do NOT use any DOM, document, or window reference.

RULE 5 — NO window.addEventListener FOR SCROLL PARALLAX
Replace the JS scroll parallax with React Native Animated:

  const scrollY = useRef(new Animated.Value(0)).current;
  const heroTranslate = scrollY.interpolate({
    inputRange: [0, 200],
    outputRange: [0, -80],
    extrapolate: 'clamp',
  });

  Use Animated.ScrollView with:
    onScroll={Animated.event(
      [{ nativeEvent: { contentOffset: { y: scrollY } } }],
      { useNativeDriver: true }
    )}
    scrollEventThrottle={16}

  Apply heroTranslate as transform on the blurred hero background View.

RULE 6 — NO HARDCODED STATS OR ACHIEVEMENTS
Every number and achievement shown must come from AppContext:
  - Recipes cooked count: from AppContext field (read Step 2 result)
  - Day streak: from AppContext field
  - Gourmet level / level number: from AppContext field
  - XP bar: xpPercent = (user.xp / user.xpToNextLevel) * 100
  - Achievements grid: .map() over the achievements array from AppContext

  WRONG:  <Text>124</Text>  or  <Text>7 🔥</Text>
  RIGHT:  <Text>{user.recipesCooked}</Text>  (using the real field name)
