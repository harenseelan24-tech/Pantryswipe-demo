# PANTRYSWIPE — COMPLETE ONBOARDING FLOW
## Master Build Prompt: Welcome → Account Creation → Pantry Setup → Discover

> This prompt covers the entire first-time user experience from splash screen to the Discover page. Paste into Replit Agent as a standalone feature build on top of the existing codebase.

---

## OVERVIEW

The onboarding is a **multi-step wizard** with a persistent progress bar at the top. Each step has a smooth horizontal slide transition (like swiping pages). The user cannot skip ahead — each step has a validation gate. The back button is always available (top left) except on Step 1.

**Total steps: 6 pages** (not 10 individual screens — combined as below)
1. Welcome — Name + Password + Household Size (all on one page)
2. Skill Level + Weekly Budget (both on one page)
3. Dietary Profile + Allergies (both on one page)
4. Your Goal (one page)
5. Cuisine Preferences (one page)
6. Pantry Setup (scan fridge / scan receipt / type / skip — one page) → Creating Account animation → Discover

  ✡️ Kosher

    "High fat, very low carb (under 20g net carbs/day)"
    → Filters: removes high-carb recipes (pasta, rice, bread, potatoes, sugary dishes); shows net carbs prominently on recipe cards; adjusts macro targets accordingly

  🍖 Paleo
    "Whole foods — no grains, legumes, dairy, or processed food"
    → Filters: removes recipes with grains (bread, pasta, rice, oats), dairy, legumes (beans, lentils, peanuts), refined sugar, processed ingredients

HOW THIS INTEGRATES:
  - dietary_restrictions stored as string array in users table: ["halal", "keto"]
  - Every recipe in the database is tagged with dietary flags
  - All recipe filtering, swipe deck, AI suggestions, and social feed recipe buttons respect these filters
  - If a posted social recipe conflicts with dietary profile: "⚠️ This recipe contains pork — not compatible with your Halal diet. See alternatives?"
  - Recipe detail ingredient list: incompatible ingredients highlighted in red with suggested swap:
      "🚫 Bacon → ✅ Swap with turkey strips or mushrooms"

## TECHNICAL NOTES

```
1. Onboarding state management:
   - Use React Context (OnboardingContext) to hold all answers during the flow
   - Only write to DB on final Step 10 (creating account)
   - If user closes app mid-onboarding → store step number + answers in localStorage
   - On re-open: restore from localStorage, ask "Welcome back! Continue where you left off?"

2. Camera integration:
   - Use navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment' } })
   - Falls back to facingMode: 'user' if environment camera unavailable
   - For receipt scan: use canvas to capture frame from video element
   - Send frame as base64 to backend → backend calls Claude Vision API
   - Never send raw camera feed to server — only single captured frames

3. Password security:
   - bcrypt with 12 salt rounds
   - zxcvbn library for password strength meter (show strength bar under password field)
   - Never store plaintext password anywhere

4. Performance:
   - Each step component lazy-loaded to minimise initial bundle
   - Wok animation: use Lottie JSON animation OR pure CSS/SVG keyframes (prefer CSS for performance)
   - Images in cuisine grid: use emoji flags only (no external image fetches)

5. Accessibility:
   - All selection cards have proper aria-selected, role="radio" or role="checkbox"
   - Progress bar has aria-valuenow and aria-valuemax
   - Skip links for screen readers on tutorial
   - Error messages linked to inputs via aria-describedby
```
